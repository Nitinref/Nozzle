# SigNoz Implementation Guide

This repo is wired for a local SigNoz demo with live customer-level AI spend attribution.

## Hackathon Checklist

- Install SigNoz with Foundry.
- Enable the SigNoz MCP server if you want conversational telemetry queries.
- Commit `casting.yaml` and the generated `casting.yaml.lock`.
- Use traces plus at least one of query/dashboards/alerts in the demo.
- Keep the deployment reproducible from a clean clone.

## 1. Self-host SigNoz

Current SigNoz docs recommend Foundry as the supported install path for Docker Compose.

1. Install Foundry:

```powershell
curl -fsSL https://signoz.io/foundry.sh | bash
```

2. Use the starter [casting.yaml](/D:/nozzle/casting.yaml).
3. Deploy SigNoz:

```powershell
foundryctl cast -f casting.yaml
```

`cast` validates the environment, generates Compose files into `pours/deployment/`, and starts the stack. `forge` is the step that writes `casting.yaml.lock`.

4. Verify:

```powershell
docker ps
```

Open `http://localhost:8080` for the UI.

5. Optional MCP server:

```yaml
spec:
  mcp:
    spec:
      enabled: true
```

Re-run `foundryctl cast -f casting.yaml`. SigNoz will expose the MCP service on port `8000`.

6. Commit the generated `casting.yaml.lock` after a successful Foundry run.

## 2. Instrument the backend

The backend in this repo already emits OpenTelemetry spans for:

- `llm.openrouter.chat`
- `tool.customer_context_lookup`

Each span includes:

- `customer_id`
- `model`
- `input_tokens`
- `output_tokens`
- `cost_usd`
- `call_type`

Environment variables:

```powershell
OTEL_SERVICE_NAME=per-customer-ai-cost-radar
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
OTEL_EXPORTER_OTLP_TRACES_ENDPOINT=http://localhost:4317
OTEL_EXPORTER_OTLP_INSECURE=true
OTEL_EXPORTER_OTLP_TRACES_INSECURE=true
```

## 3. Verify traces in SigNoz

1. Open `APM` or `Traces Explorer`.
2. Filter on service `per-customer-ai-cost-radar`.
3. Search for `customer_id = 'acme'`.
4. Confirm nested spans for model calls and tool lookups.

Useful trace names:

- `llm.openrouter.chat`
- `tool.customer_context_lookup`

## 4. Consume the data

This repo exposes live aggregation endpoints:

- `GET /customers/summary`
- `GET /customers/{customer_id}/alerts`

These endpoints query SigNoz data, compute rolling baselines, and flag customer spend spikes when current spend exceeds `3x` the baseline.

## 5. Demo checklist

- Commit `casting.yaml`
- Commit `casting.yaml.lock`
- Commit the backend and frontend source
- Keep `pours/` generated by Foundry out of hand edits
- Document any AI assistance used in the project README
